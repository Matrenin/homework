	========= Обработка событий ============

	При использовании модификаторов порядок имеет значение, потому что код генерируется в том же порядке. 

	========= Работа с формами ============

	v-model игнорирует начальное значение атрибутов value, checked или selected на любых элементах форм. Данные экземпляра Vue всегда считаются источником истины. Начальное значение необходимо объявить на стороне JavaScript, внутри опции data компонента.

	Интерполяция внутри textarea (<textarea>{{text}}</textarea>) не будет работать. Используйте вместо неё директиву v-model.

	Если начальное значение выражения v-model не соответствует ни одному из вариантов списка, элемент <select> будет отображаться в «невыбранном» состоянии. Рекомендуется предоставлять отключённый disabled-вариант выбора с пустым значением value.

	========= Основы компонентов ============

	Компонент – логически независимая часть приложения, которая отвечает за обработку и визуализацию конкретной задачи.
