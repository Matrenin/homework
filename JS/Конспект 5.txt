	history.pushState() - добавляет новое состояние в историю браузера.

	Событие hashchange генерируется когда изменяется идентификатор фрагмента URL (т.е. часть URL следующая за символом #, включая сам символ #).

	Событие popstate вызывается, когда изменяется активная запись истории. Если изменение записи истории было вызвано методом history.pushState() или history.replaceState(), то состояние события popstate будет содержать state копии входящего в историю объекта.

	В консоли прописываем
vue add unit-jest - добавляет модуль тестирования(видео урок). 
npm install --save-dev @vue/test-utils - устанавливаем библиотеку(методичка).

	Jest - наиболее функциональная утилита, которая содержит множество настроек для запуска тестов.

npm install --save-dev jest vue-jest - устанавливаем jest.

	Существует три уровня тестирования:
1. Юнит тесты.
2. Серверные и интеграционные тесты.
3. Тесты пользовательского интерфейся.

	Главные принципы тестирования - написать множество мелких (unit) тестов, написать несколько более общих (интеграционных) тестов, и напоследок написать небольшое число высокоуровневых тестов пользовательского интерфейса.

	Vue-jest, устанавливается вместе с jest - с его помощью мы можем научить раннер обрабатывать однофайловые компоненты (файлы с расширением .vue).
