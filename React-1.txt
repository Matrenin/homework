	Компоненты - по сути, просто функция или класс javascript(класс должен наследоваться от React.Component и реализовывать метод render). Соответственно, компоненты делятся на функциональные и классовые.

	Добавление корневого компонента в DOM осуществляется с помощь метода ReactDOM.render - этот метода принимает компонент, который необходимо добавить, и элемент DOM, в который следует добавить компонет. Как правило, в приложении вызов этого метода происходит лишь однажды - для корневого компонента, а он в свою очередь, рендерит остальные компоненты.

	JSX — расширение синтаксиса JavaScript. Этот синтаксис выглядит как язык шаблонов, но наделён всеми языковыми возможностями JavaScript. Элементам DOM в JSX можно ставить атрибуты аналогично тому, как это делается в HTML (за исключением атрибута class - так как это слово является в js ключевым, вместо него указывается атрибут className).
	
	Когда React встречает подобный элемент, он собирает все JSX-атрибуты и дочерние элементы в один объект и передаёт их нашему компоненту. Этот объект называется «пропсы» (props).

	Компонент никогда не должен что-то записывать в свои пропсы — вне зависимости от того, функциональный он или классовый. React-компоненты обязаны вести себя как чистые функции по отношению к своим пропсам. Чистые функции - это функции, которые не меняют свои входные данные и предсказуемо возвращают один и тот же результат для одинаковых аргументов.

	«Состояние» очень похоже на пропсы, отличие в том, что состояние контролируется и доступно только конкретному компоненту.
	
	В функциональных компонентах не существует методов цикла, однако аналогичное поведение достигается использованием хука useEffect. Он принимает два аргумента - коллбэк и массив зависимостей. Коллбэк useEffect будет выполнен: при монтировании компонента / при изменении одной из зависимости. Если указать вторым аргументом пустой массив - функция будет выполнена один раз, только после монтирования компонента (аналогично componentDidMount).
